# Flock — Final Elite-Readiness Report

## 1. Executive verdict

**Overall grade: B (B+ on architecture, held down to B by a broken prod image build, two governance gaps that are claimed-done, and supervision state that evaporates on restart).**

Flock's moat is real and unique: it is the only tool in its class that pairs a **browser UI** with **real multi-node remote execution over SSH** and **fleet-level supervision** (cross-agent attention inbox, outcome-grouped Mission Control, spatial topology, compare/race). No single-machine competitor (Synara, HiveTerm, Herdr, Superset) can offer the multi-node lens, and Flock makes it _legible_ rather than just claimed. The engineering underneath is genuinely strong — elite secret/crypto handling, a correctly-upheld "Postgres never on the hot path" contract, panic-safe Go concurrency, and an enviable single-source-of-truth type discipline. It is **not shippable today** because of one critical defect (the orchestrator Docker image cannot build), and it is **not yet decisively elite** because its headline governance features (tool-call gating, cost budgets) are positioned/tracked as done but do not exist, and key supervision state (teams graph, reviewed flag) is non-durable.

## 2. What's genuinely elite

- **Security core.** AES-256-GCM secret store (fresh 96-bit nonce/op, auth-tag verified, key-version rotation, decrypt-time audit), argon2id at OWASP cost with timing-flattened login, SSH host-key TOFU pinning with MITM rejection on every reconnect, default-deny surface guard, anti-CSWSH + per-session ownership on WS, and clean secret hygiene (only public fingerprints ever logged).
- **Orchestrator hot-path discipline (NFR-PERF1).** `StatusMap.set()` mutates + fans out synchronously then defers writes via `queueMicrotask`; `WriteBehindEventQueue` is bounded, retrying, sheds-oldest on a wedged DB; the hook endpoint is genuinely DB-free.
- **agentd concurrency.** Panic-recover on _every_ goroutine entry point; `broadcast()` is non-blocking fan-out under the same mutex as `finalize()`'s `close(ch)` (documented send-on-closed-channel safety), verified under `-race`; Landlock sandbox applied in a short-lived re-exec helper, ABI-masked, fails CLOSED.
- **Shared contracts.** Every value enum lives once and is asserted against the DB schema by a CI guardrail (`schema.test.ts`); hook payloads use `.passthrough()` + both casings so version-skewed agents never get dropped on the hot path; `agentEventToStatus` uses an exhaustive no-default switch (new kind = compile error).
- **Agent layer as config-as-data.** One exhaustive `AGENT_CAPS` `Record<AgentType,...>` drives argv/flags/daemon-kind/initial-status; adding a CLI agent is a one-row edit; translators are pure, schema-validated, fixture-tested.
- **Web/terminal craft.** Resize-dedupe, fit guard, reset-on-reconnect, deferred dispose; pure router-sync keeps terminals mounted; store is UI-only with queries owning server data.
- **Supervision IA.** Mission Control groups by _what the human must do_ (Needs you / Ready to review / Uncommitted / Teams / Working / Idle), dedupes spawned children, degrades to Quick-start when calm. The cross-agent **AttentionInbox** is the single highest-value primitive and is cleanly built (pure selector, fleet-wide, always visible).
- **Token system.** `tokens.ts` single source mirrored into `theme.css` + `tailwind.config.cjs`, drift-guarded by contract tests, RGB-channel vars so `/opacity` works, correct first-paint via `prefers-color-scheme`.

## 7. Docker/deploy & configurability

**Verdict: thoughtful surface, not yet shippable (C).** Strong: Caddy-only ingress with spec-annotated network posture, Docker-secret-file support with env fallback, cached dep layers, non-root runtime, atomic idempotent agentd bootstrap. Blocking/important gaps:

- **Build break (P0):** Go 1.22 stage vs Go 1.25 `go.mod` — image won't build (§3).
- **Perpetually-unhealthy container:** `/healthz` probe vs `/health` route.
- **`FLOCK_AGENTD_VERSION` pin fragility:** the single source of truth (`agentd/VERSION` = 0.2.39-dev) is shadowed by a stale compose default (`0.2.8-dev`, line 119) that `resolveAgentdVersion()` prioritizes _above_ the COPYed VERSION file. A default deploy therefore injects the old version → `installed != expected` on every connect → perpetual daemon re-ship/relaunch that **kills live sessions**. Fix: drop the compose literal (resolve from the COPYed `agentd/VERSION`), make the code fallback fail-loud if absent, and add a CI assertion `constant == agentd/VERSION`.
- **Configurability debt:** model/pricing table, several orchestrator retry/TTL/backoff literals, and agentd tuning knobs (scrollback, activity window, shutdown grace) are compile-time constants — promote the high-value ones to env with current values as defaults.
- Pin web/nginx bases; pin/verify the `curl|bash` agent installs; delete dead `SESSION_SECRET`.

## 8. Competitive position & the highest-leverage moves

Flock's thesis — _supervise AND direct a multi-agent fleet across many machines from a browser_ — is coherent and the multi-node moat is genuine and well-surfaced. But several "elite" features are thinner than the positioning: governance is the weakest leg (no gating engine, no budgets — both effectively claimed-done), the three fleet lenses overlap (and Terminal is mislabeled), and team/durable state is ephemeral.

**The 4 moves to be decisively elite:**

1. **Close the governance gap (the wedge vs Mission Control's Aegis).** Ship ONE agent (OpenCode) with a true structured approve/deny round-trip + an audit record of _what_ was approved; relabel the blind y/n buttons until then.
2. **Build minimal cost/policy governance.** Per-project/per-session spend cap → server-side over-budget pause + alert. This is the named Phase-2 moat and is currently display-only.
3. **Make supervision state server-durable & team-shared.** Persist the spawn/handoff teams graph (table + boot rehydrate) and promote `reviewed` to a server field. This is what makes the "team co-operates a fleet" story true.
4. **De-duplicate the fleet lenses.** Earn the "Terminal" name with a real PTY tail, or collapse to two lenses (Triage + Topology) so the unique SpatialFleet stands out instead of being diluted by a restyled card grid.

(Fast-follow once the above land: P2 steer/inject/queue composer — converts Flock from monitor to controller; M10 MCP/skills surface.)
